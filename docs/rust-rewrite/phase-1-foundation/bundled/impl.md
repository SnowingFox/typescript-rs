# bundled: 实现方案（impl.md）

**crate**：`tsgo_bundled`　**目标**：把 TypeScript 自带的 `lib.*.d.ts` 标准库文件**内嵌进二进制**，并通过一个虚拟文件系统（`bundled:///` 方案）暴露给编译器/语言服务，使发布的单文件可执行程序无需额外携带 stdlib。
**依赖（crate）**：`tsgo_vfs`（`vfs::FS` trait）、`tsgo_tspath`（路径规范化）。测试还用到 `tsgo_vfs::osvfs`。
**Go 源**：`internal/bundled/`（6 个非测试文件 + `libs/` 子目录 110 个 `.d.ts` 数据文件）。

## 这个包是什么（业务说明）

TypeScript 编译器在类型检查任何程序前，都要先加载一组标准库声明文件（`lib.es5.d.ts`、`lib.dom.d.ts`、`lib.esnext.d.ts` 等，共 110 个，最大的 `lib.dom.d.ts` 约 2.3MB）。Strada（TS 官方实现）把这些文件随 `tsc` 一起分发；typescript-go 把它们用 Go 的 `//go:embed` **编译进二进制**，运行时通过一个伪 FS 暴露在 `bundled:///libs/` 路径下，从而做到"零外部依赖的单文件 tsc"。

本包提供三件事：
1. **内嵌数据**：`embed_generated.go` 把每个 lib 文件 `//go:embed` 成一个 `string` 常量，汇总成 `embeddedContents map[string]string`。
2. **伪文件系统包装器**：`WrapFS(fs)` 返回一个 `vfs.FS`，凡是以 `bundled:///` 开头的路径都路由到内嵌内容，其余转发给真实 FS。`LibPath()` 返回 `bundled:///libs`。
3. **代码生成器**：`generate.go`（`//go:build ignore`，由 `go generate` 调用）从 TS 子模块的 `src/lib/libs.json` 读取 lib 清单，拼接版权头后写出 `libs/*.d.ts`，并生成 `libs_generated.go`（`LibNames`）与 `embed_generated.go`（embed 指令 + map）。

它处于 **Phase 9**，因为 cmd/tsgo（入口）、execute（CLI）、lsp、api 都通过 `bundled.WrapFS` + `bundled.LibPath` 装配文件系统；它本身只依赖 `vfs`/`tspath`，是 phase 内最叶子的包。

另有一个编译期开关：`embedded` 常量（`Embedded`）。`!noembed`（默认）走内嵌；`noembed`（构建标签）则改为从可执行文件所在目录读取磁盘上的 lib 文件——用于不希望把 2MB+ 数据塞进二进制的开发构建。

## 所有权 / 类型映射（本包关键决策）

通用规则见 PORTING.md §3。本包特有决策：

| Go 构造 | Rust 表示 | 说明 |
|---|---|---|
| `//go:embed libs/lib.X.d.ts` + `var libs_lib_X_d_ts string` | `include_str!("libs/lib.X.d.ts")` | 编译期把文本嵌入只读数据段，等价 go:embed string。文件大但都是文本（UTF-8）。 |
| `embeddedContents map[string]string` | `static EMBEDDED_CONTENTS: phf::Map<&str, &str>` 或 `LazyLock<FxHashMap<&'static str, &'static str>>` | key=`"libs/lib.X.d.ts"`，value=`include_str!`。用 `phf`（编译期完美哈希）零运行时构建成本；退而求其次 `LazyLock<FxHashMap>`。**只读、键固定**，不需要 IndexMap。 |
| `LibNames []string`（按名排序） | `pub static LIB_NAMES: &[&str]`（生成、按名排序） | 顺序必须与 `WalkDir` 遍历顺序一致（见 `TestEmbeddedLibs`）。 |
| `libsEntries []fs.DirEntry` | `static LIBS_ENTRIES: &[FileInfo]`（生成） | 用于 `walk_dir`，每项含 name + size。 |
| `const embedded = true/false`（构建标签 `noembed`） | `#[cfg(feature = "embed")]` / `#[cfg(not(feature = "embed"))]` + `pub const EMBEDDED: bool` | Go build tag → Cargo feature `embed`（默认开）。两套 `wrap_fs`/`lib_path`/`is_bundled` 实现按 feature 分发。 |
| `wrappedFS` 实现 `vfs.FS` 18 个方法 | `struct WrappedFs<F: FS> { inner: F }` impl `FS` | 直接实现 `vfs::FS` trait（**不**绕道 `std::io::fs`，与 Go 注释一致——`vfs::FS` 以 `String` 为内容单位，避免 `[]byte` 来回拷贝）。 |
| `splitPath` (`strings.CutPrefix`) | `path.strip_prefix("bundled:///")` → `Option<&str>` | |
| `sync.OnceValue` (`bundledSourceDir`/`testingLibPath`/`libPath`) | `LazyLock<String>` 或 `OnceLock<String>` | |
| `runtime.Caller(0)` 取源文件目录（仅测试用） | `env!("CARGO_MANIFEST_DIR")` 拼 `libs` | Rust 在编译期就知道 crate 根目录，比 `runtime.Caller` 更稳。 |
| `os.Executable()` + `Realpath`（noembed） | `std::env::current_exe()` + `realpath` | |
| `fileInfo`（实现 `fs.FileInfo`+`fs.DirEntry`） | `struct FileInfo` impl `vfs::FileInfo` + `vfs::DirEntry` | mode/name/size 三字段。 |
| `fs.SkipAll` / `fs.SkipDir`（WalkDir 控制流） | `vfs::WalkDir` 的控制枚举（P1 vfs 决定） | `walk_dir` 的 early-stop 语义需与 vfs 抽象一致。 |
| panic on write to embedded FS | `panic!("cannot write to embedded file system")` | `write_file`/`append_file`/`remove`/`chtimes` 对 `bundled:///` 路径 panic，与 Go 一致。 |

### 内嵌方案（重点，cmd/tsgo 发布形态的命门）

Go 用三条流水线：`go:embed`（数据进二进制）+ 构建标签（embed/noembed 切换）+ `go generate`（从 TS 子模块再生成）。Rust 等价方案：

**A. 数据内嵌（对应 `embed_generated.go`，默认 `feature = "embed"`）**
- 生成一个**已提交**的 `internal/bundled/embed_generated.rs`，内容形如：
  ```rust
  // Code generated by generate; DO NOT EDIT.
  pub(crate) static EMBEDDED_CONTENTS: phf::Map<&'static str, &'static str> = phf::phf_map! {
      "libs/lib.d.ts" => include_str!("libs/lib.d.ts"),
      "libs/lib.es5.d.ts" => include_str!("libs/lib.es5.d.ts"),
      // ... 110 条，与 LibNames 同序
  };
  pub(crate) static LIBS_ENTRIES: &[FileInfo] = &[
      FileInfo::file("lib.d.ts", include_str!("libs/lib.d.ts").len() as i64),
      // ...
  ];
  ```
  `include_str!` 在 `rustc` 链接期把文本并入 `.rodata`，与 `go:embed` 完全同语义（编译进二进制、运行时零 I/O、`&'static str` 借用）。
- 备选：`include_bytes!`（若想避免 UTF-8 校验开销）→ 需在读取处 `str::from_utf8`。lib 文件保证 UTF-8，故首选 `include_str!`。
- `rust-embed` crate 也能做，但它默认走 `fs::FS` 风格、运行时构建索引，且 debug 模式回退磁盘——与本包"直接实现 vfs::FS、避免拷贝"的诉求不符，**不采用**。

**B. 编译期开关（对应 `//go:build noembed` / `embed.go` vs `noembed.go`）**
- Cargo feature `embed`（默认在 `[features] default = ["embed"]`）。
- `#[cfg(feature = "embed")] mod embed;`（含 `EMBEDDED = true`、`bundled:///` 方案、`WrappedFs`）。
- `#[cfg(not(feature = "embed"))] mod noembed;`（含 `EMBEDDED = false`、`wrap_fs` 直接返回原 FS、`lib_path()` 用可执行文件目录、`is_bundled` 恒 `false`）。
- `lib.rs` 重导出统一 API：`wrap_fs` / `lib_path` / `is_bundled` / `EMBEDDED`。

**C. 再生成器（对应 `generate.go` + `//go:generate`）**
- 一个 **xtask 风格的二进制** `internal/bundled/src/bin/generate.rs`（或 workspace 级 `xtask generate-libs`），手动运行（等价 `go generate`），它：
  1. 读 `repo::typescript_submodule_path()/src/lib/libs.json`（JSONC，需先 strip 注释 + 尾逗号——见 `stripJSONC`）。
  2. 读 `scripts/CopyrightNotice.txt`，CRLF→LF。
  3. 对每个 lib：拼接版权头 + `\n` + 源 `.d.ts`（CRLF→LF），写 `libs/<target>`。
  4. 按 `target` 名排序后，生成 `libs_generated.rs`（`LIB_NAMES`）与 `embed_generated.rs`（`include_str!` 映射 + `LIBS_ENTRIES`）。
- **不**用 `build.rs` 自动跑：与 Go 的 `go:generate`（手动、产物入库）保持一致，避免每次构建都去碰 TS 子模块。`build.rs` 仅作为备选在 impl 里注明。

> 偏离记录：`bundled:///` 是自定义 URL scheme，不是真实路径；Rust 侧 `WrappedFs` 必须对所有 `vfs::FS` 方法做 scheme 前缀判定后路由，结构与 Go `wrappedFS` 1:1。

## 与 Go 的已知偏离（divergence）

- **构建标签 → Cargo feature**：`//go:build noembed` 映射为 `#[cfg(not(feature="embed"))]`。语义等价，但激活方式不同（`-tags noembed` → `--no-default-features`）。
- **`runtime.Caller(0)` → `CARGO_MANIFEST_DIR`**：Rust 编译期常量更可靠，但要求 `libs/` 与 crate 根的相对位置不变（与 Go 假设一致）。
- **内嵌 map 用 `phf`**：键集合固定、编译期已知，比 Go 运行时建 `map` 更省；若不引 `phf` 则 `LazyLock<FxHashMap>`，行为一致。
- **`WalkDir` 控制流（SkipAll/SkipDir）** 依赖 P1 `tsgo_vfs` 对 walk 回调返回值的抽象；若 P1 用 `ControlFlow`/枚举，本包按其约定适配。

## 转交 / 推迟（DEFER）

- `tsgo_vfs::FS` trait 与 `WalkDir`/`FileInfo`/`DirEntry`/`osvfs` 来自 **P1**；本包实现 `WrappedFs` 时 `// DEFER(phase-1) blocked-by: tsgo_vfs` 仅指依赖就绪即可，本包逻辑不推迟。
- `repo::typescript_submodule_path()`（generate 用）来自 **P1 `tsgo_repo`**。
- 实际 110 个 `.d.ts` 内容文件不翻译，作为内嵌数据资产长期保留。
