### 关键路径（串行瓶颈链）

整条仓库的 **最长串行链**（其余 crate 应尽量与此链并行消化）：

```
ast → scanner → parser → tsoptions → printer → checker★★ → transformers → compiler → ls★ → project → execute → cmd/tsgo
```

---

## 可并行 vs 串行

| 维度 | 说明 |
|------|------|
| **并行单位** | **Crate**（一个 `internal/<pkg>` 对应一个 `tsgo_<pkg>`），不是「整个 phase 一次性做完」。 |
| **波次关系** | 波1…波10 是 **依赖解锁顺序**；实际推进时各波可 **交错**（例如波1 未全绿时，已满足的波2 crate 可先开工）。 |
| **最大并行度** | **波1 ×6**（scanner、evaluator、nodebuilder、pseudochecker、outputpaths、packagejson）；**波2 ×7**（parser、astnav、binder、sourcemap、tracing、diagnosticwriter、module）。 |
| **串行主轴** | 见上图 **关键路径**：AST 基石 → 词法/语法/options/emit 前置 → checker → emit/编译 → 语言服务 → 工程/LSP → CLI。 |
| **单 owner 巨 crate** | **checker**（波5，★★，含 4a–4k 子阶段）、**ls**（波7 根 crate，★，子 crate 先并行再汇入）。 |

**实践含义**：人力应按 crate 切片并行；排期评审时以关键路径上的 ★ 节点为里程碑，而非「P4 全部完成」这种粗粒度 gate。

以下包的 **生产构建依赖** 已全部落在 P1 ✓，不等待 `ast` 或后续波次：

可在波1 同期或更早独立推进（与 README 中 P7 文档归类不冲突——文档 phase 为实施主题，本路线图按 **真实 build edge** 排波前）。

## 依赖依据（关键构建边）

以下条目均来自 **非 `*_test.go` 构建边**（与 [README.md](./README.md) gate D6 口径一致）：

| 依赖方 | 依赖项 | 备注 |
|--------|--------|------|
| `binder` | `scanner` **仅** | **不**依赖 `parser`（波2 可与 parser 并行） |
| `module` | `packagejson`（波1）+ `vfs`（P1） | 与 parser 无直接构建边 |
| `tsoptions` | `module` + `outputpaths` + `parser` + `scanner` | 关键路径上承接 parser |
| `symlinks` | `module` | 波3 与 tsoptions 可并行（不同 gate） |
| `modulespecifiers` | `module`、`outputpaths`、`packagejson`、`symlinks`、`tsoptions` | 波4 |
| `printer` | `binder`、`evaluator`、`nodebuilder`、`sourcemap`、`tsoptions` | 关键路径上在 tsoptions 之后 |
| `checker` | 上述波1–4 全部相关前置（**all** 构建前置汇聚） | 波5 单点；最难 |
| `transformers` | `checker` 等 P4/P5 前置 | 波6 串行在 checker 之后 |
| `compiler` | `transformers` 链 | 波6 |
| `ls`（根） | `format` + `ls/*` 子 crate 等 | 波7 大单点 |
| `project` | `ls` 侧前置（含 `ls/autoimport` 等） | 波8 |
| `execute` | 编译/工程链前置 | 波9 |
| `testutil` → `testrunner` / `fourslash` / conformance | 测试设施 | 波10；生产 crate 以 dev-dep 引用，不反向约束波前 |

## 阶段对照（简表）

| 标记 | 波次 / Phase | Crates |
|------|----------------|--------|

| 波1 | ast 后 ×6 | scanner, evaluator, nodebuilder, pseudochecker, outputpaths, packagejson |
| 拉前 | P1 即可 | lsproto, project/dirty, project/logging |
| 波2 | scanner 后 ×7 | parser, astnav, binder, sourcemap, tracing, diagnosticwriter, module |
| 波3 | ×2 | symlinks, tsoptions |
| 波4 | ×2 | modulespecifiers, printer |
| 波5 ★★ | ×1 | checker（4a–4k） |
| 波6 | 串行链 | transformers → compiler |
